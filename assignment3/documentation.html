Phones API Collection

GET : Index

The GET method requests a representation of the specified resource. The GET Index requests a representation of the whole data in the web server. The results are paginated. To get to the next page, insert query parameter `page` to specify which page you want to go.

URL

http://localhost:3000

Information

Response formats: JSON

Parameters

Name Param Type Required Description Example
page Query Required Page requested 1

~EMPTY~

GET : Show

The GET method requests a representation of the specified resource. The GET Show requests a representation of a specific item with a specific id paramater.

URL

http://localhost:3000

Information

Response formats: JSON

Parameters

Name Param Type Required Description Example
phone_id URL Required ID of the phone requested 2

Example Request

curl --location --request GET 'http://localhost:3000/2'

POST : Create

When this route is triggered a new instance of Phone is added to the database. Therefore creating a new one.

URL

http://localhost:3000

Information

Response formats: JSON

Parameters

Name Param Type Required Description Example
phone_id URL Required ID of the phone requested 2
name Body Required Name of the phone Iphone X
os Body Required Operating System of the phone Android
model Body Required Model of the phone Galaxy S5
screensize Body Required Screensize of the phone 2048

Example Request

curl --location --request POST 'http://localhost:3000' \ --data-raw '{ "os":"OSX", "brand": "Apple", "model" : "Iphone X", "image": "https://upload.wikimedia.org/wikipedia/commons/thumb/c/c4/Fairphone_3_modules_on_display.jpg/320px-Fairphone_3_modules_on_display.jpg", "screensize":5.12 }'

Example Response

STATUSCODE:201
Body: {
"id": 4,
"brand": "Apple",
"model": "IPhone X",
"os": "OSX",
"image": "https://upload.wikimedia.org/wikipedia/commons/thumb/c/c4/Fairphone_3_modules_on_display.jpg/320px-Fairphone_3_modules_on_display.jpg",
"screensize": 5.12
}


PUT : Updates the Phone

This method updates the requested Phone model with the fields sent in the body. Requested phone is determined by the phone_id in the url parameter

URL

http://localhost:3000/:phone_id

Information

Response formats: JSON

Parameters

Name Param Type Required Description Example
phone_id URL Required ID of the phone requested 2
name Body Optional Name of the phone Iphone X
os Body Optional Operating System of the phone Android
model Body Optional Model of the phone Galaxy S5
screensize Body Optional Screensize of the phone 2

Example Request

curl --location --request PUT 'http://localhost:3000/4' \ --data-raw '{ "model": "Iphone X" }'

Example Response

STATUSCODE:200
Body: {
"id": 4,
"brand": "Samsung",
"model": "IPhone X",
"os": "LINUX",
"image": "https://upload.wikimedia.org/wikipedia/commons/thumb/c/c4/Fairphone_3_modules_on_display.jpg/320px-Fairphone_3_modules_on_display.jpg",
"screensize": 2
}
